🔹 Endereço de Rede

👉 O primeiro IP da rede (todos os bits de host = 0).
Ele identifica a rede em si, não uma máquina.
Exemplo:
Rede: 192.168.1.0/24
Endereço de rede = 192.168.1.0
Serve para representar a rede 192.168.1.0/24 inteira.
🔹 Endereço de Broadcast

👉 O último IP da rede (todos os bits de host = 1).
Ele é usado para enviar pacotes para todos os hosts da rede ao mesmo tempo.
Exemplo:
Rede: 192.168.1.0/24
Endereço de broadcast = 192.168.1.255
Se você mandar algo para esse IP, todas as máquinas na rede recebem.

🔹 Exemplo prático
Rede: 192.168.1.0/24
Endereço de rede: 192.168.1.0
Endereço de broadcast: 192.168.1.255
Faixa de hosts válidos: 192.168.1.1 → 192.168.1.254

🔹 Como interpretar
/24 → 256 endereços totais (1 rede + 1 broadcast + 254 hosts).
/25 → divide a rede em duas metades (128 endereços cada).
/30 → usado em links ponto a ponto (só 2 hosts possíveis).
/16 → rede bem maior (65.534 hosts possíveis).

🔹 1. Endereço IP
Um IP é composto por 32 bits (IPv4), dividido em 4 octetos (ex: 192.168.1.10).
Cada octeto vai de 0 a 255.

🔹 2. Máscara de Sub-rede (Subnet Mask)

Define qual parte do IP é rede e qual parte é host.
Exemplo:
IP: 192.168.1.10
Máscara: 255.255.255.0 (ou /24)
Significa que os 3 primeiros octetos (192.168.1) são a rede e o último octeto é para hosts.

🔹 3. Gateway
É o ponto de saída de uma rede.
Se a máquina quer falar com alguém fora da sua sub-rede, ela precisa enviar os pacotes para o gateway,
que encaminha para outras redes.

🔹 4. Roteamento

Se duas máquinas não estão na mesma sub-rede, você precisa de roteadores ou gateways para conectar.
Cada roteador precisa ter interfaces configuradas corretamente para “falar” com as duas redes.

🔹 5. Regras de Conectividade
No NetPractice, para duas máquinas se comunicarem, você precisa verificar:
Estão na mesma rede?
Compare IP + máscara → se sim, podem se comunicar diretamente.
Se não estão na mesma rede:
Precisa de um roteador/gateway entre elas.
Gateway está configurado?
A máquina deve ter o gateway certo configurado para sair da rede.


🔹 6. Notação CIDR

Forma compacta de escrever máscara.
Exemplo:
192.168.1.0/24 = rede 192.168.1.x, com máscara 255.255.255.0.
10.0.0.0/16 = rede 10.0.x.x, com máscara 255.255.0.0.
